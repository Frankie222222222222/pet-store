# 项目概览

- 项目名称：泡泡爪印宠物洗护店。
- 这是一个宠物洗护门店的中文宣传与预约演示网站。
- 项目是纯静态单页：没有前端框架、构建工具、数据库和真正的业务后端。
- 页面内容、CSS 样式和 JavaScript 交互都集中在 `index.html` 中。
- `server.js` 是用 Node.js 内置模块编写的静态文件服务器，不依赖第三方 npm 包。

# 运行方式

1. 确保电脑已安装 Node.js。
2. 在项目根目录运行 `npm start`。
3. 浏览器访问 `http://127.0.0.1:3000`。

补充说明：

- `npm start` 实际执行的是 `node server.js`。
- 服务固定监听 `127.0.0.1:3000`。
- 当前没有可用的自动化测试；`npm test` 只会提示“没有测试”并以失败状态退出。
- 本项目没有 npm 依赖，所以正常情况下不需要执行 `npm install`。

# 文件与目录职责

- `index.html`：完整网页，包含 HTML 页面结构、内嵌 CSS 和内嵌 JavaScript。
- `server.js`：静态资源服务器；根路径 `/` 返回 `index.html`，其他路径按文件名读取，并处理常见资源的 Content-Type。
- `package.json`：项目元数据和 `start`、`test` 命令。
- `assets/`：项目本地图片资源。
- `AGENTS.md`：面向后续协作者的项目说明和协作约定。

`assets/` 中当前真正被页面使用的图片：

- `hero-paw-spa.png`：首页轮播第 1 张图。
- `hero-nova-pets.jpg`：首页轮播第 2 张图。
- `hero-luxury-salon-2.jpeg`：首页轮播第 3 张图。
- `store-location-map.png`：预约区域中的门店位置示意图。

`hero-luxury-salon.jpeg`、`hero-luxury-salon-3.jpeg`、`hero-luxury-salon-4.jpeg` 当前没有在页面中引用，删除或替换前应再次搜索确认。

# 页面内容结构

页面从上到下依次为：

1. 顶部粘性导航栏：品牌、洗护服务、护理流程、套餐价格、预约到店。
2. 首页大图轮播：3 张本地图片，带左右按钮、页码按钮和主行动按钮。
3. 店铺亮点数据：12 年经验、1 对 1 工作台、4.9 分评价、可视化洗护。
4. 洗护服务：基础洗护、猫咪专护、造型修剪。
5. 护理流程：到店评估、分区洗护、吹干梳理、交付反馈。
6. 套餐价格：轻松清爽洗、全身精致护、美容造型。
7. 顾客评价轮播：共 6 条评价。
8. 预约区域：营业信息、联系方式、位置图和预约表单。
9. 页脚版权信息。

导航使用 `#services`、`#process`、`#pricing`、`#booking` 等锚点在当前页面内跳转，没有多页面路由。

# 样式设计规律

- 整体风格是米白背景、深青色文字、薄荷绿和珊瑚橙点缀。
- 颜色、阴影等基础值统一定义在 CSS 的 `:root` 变量中，如 `--teal`、`--coral`、`--paper`、`--shadow`。调整全站主题时优先修改这些变量。
- 主要内容最大宽度为 `1160px`，并通过 `calc(100% - 32px)` 给小屏幕保留左右空白。
- 卡片、按钮和图片大多使用 `8px` 圆角，修改时尽量保持统一。
- `880px` 以下切换成单列布局并隐藏中间导航链接；`560px` 以下进一步缩小字号、间距和按钮。
- 页面支持 `prefers-reduced-motion`，当用户的系统设置为“减少动画”时会关闭或减弱动画。新增动画时也要照顾这个设置。
- 页面字体优先使用微软雅黑和苹方，没有额外加载网络字体。

用大白话理解响应式样式：电脑屏幕宽时内容可以横着排，手机屏幕窄时会改成一块一块竖着排，这主要由文件中的两个 `@media` 区域控制。

# JavaScript 交互

## 首页轮播

- 通过给当前 `.hero-slide` 和 `.hero-dot` 添加 `is-active` 类来切换显示状态。
- 默认每 4.8 秒自动切换。
- 点击左右箭头或页码会切换图片，并重新开始计时。

## 顾客评价轮播

- 宽屏每页显示 2 条评价，`880px` 以下每页显示 1 条。
- 默认每 5.2 秒自动翻页。
- 鼠标移入或键盘焦点进入评价区时暂停，离开后继续。
- 浏览器标签页隐藏或系统要求减少动画时不会继续自动播放。
- 改变窗口宽度后会重新计算页数和页码按钮。

## 预约表单

- 期望到店时间使用 `datetime-local`，步长为 30 分钟。
- JavaScript 会把可选最早时间更新为当前时间，防止选择过去时间。
- 提交时调用 `preventDefault()`，只在页面上显示“预约已记录”的提示。
- 预约数据不会发送到 `server.js`，不会保存到文件或数据库，刷新页面后就消失。
- 当前只有到店时间是必填项；姓名、电话等字段尚未加 `required`，也没有手机号格式校验。

# 图片与网络依赖

- 首页轮播图和门店地图使用本地 `assets/` 文件，可以离线加载。
- 服务图片、流程背景、预约背景和顾客头像来自 `images.unsplash.com`。
- 无网络或 Unsplash 无法访问时，这些远程图片会显示失败，但页面文字和本地图片仍可使用。
- 如果网站要稳定部署，建议以后把必要的远程图片下载到 `assets/`，再改为本地相对路径。

# 静态服务器行为

- `server.js` 仅支持读取并返回项目目录内的文件。
- 找不到文件时返回 `404 Not Found`。
- URL 无法解析时返回 `400 Bad Request`。
- 代码会检查解析后的路径，阻止访问项目目录之外的文件，并返回 `403 Forbidden`。
- 服务器没有 API、登录、预约处理、日志存储或数据库连接。

# 修改约定

- 所有文本文件继续使用 UTF-8 编码，避免中文乱码。
- 页面目前采用单文件结构，小改动应沿用现有写法；除非功能明显变复杂，不要急着引入框架或拆成很多文件。
- 新增页面区块时，优先沿用现有的宽度、颜色变量、卡片圆角和响应式断点。
- 新增交互时要兼顾鼠标、键盘和手机操作，并保留合适的 `aria-label`。
- 修改类名时要同时检查 HTML、CSS 和 JavaScript，因为三者都在依赖这些类名。
- 替换或删除图片前，先搜索图片文件名，确认没有 HTML 或 CSS 引用。
- 不要把预约提示误认为真正提交成功；在接入后端 API 前，应一直明确它只是前端演示。

# 验证清单

修改后至少检查以下内容：

1. 运行 `npm start` 后，访问 `http://127.0.0.1:3000` 能正常打开首页。
2. 浏览器控制台没有 JavaScript 报错。
3. 首页轮播的自动切换、左右按钮和页码按钮都能使用。
4. 顾客评价在电脑宽度和手机宽度下都能正确翻页。
5. 导航链接能跳到对应区域。
6. 预约时间不能选择过去时间，提交后能出现中文提示。
7. 分别查看宽屏、`880px` 以下和 `560px` 以下的布局，确认没有横向溢出或文字遮挡。
8. 本地图片路径没有失效；联网时远程图片可以加载。
